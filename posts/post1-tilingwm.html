I hate Desktop Enviroments (on Tiling Window Managers) [cite: 1]

I hate desktop environments. [cite: 1] I won't say they're not useful, I won't say we don't need them. But I hate them either way. [cite: 2] If I can avoid using them I will. The only time I genuinely use them is when playing video games (which nowadays is basically never) or when editing videos / images. [cite: 3] For anything else I do I prefer using a tiling window manager. [cite: 4] I like using all kinds of Operating System, whether it's GNU/Linux, Windows, BSD, MacOS, I like using them all, as long as they properly support Tiling Window Managers. [cite: 5] MacOS, for example, has made using tiling window managers more difficult over the years. [cite: 6] This genuinely caused me to dislike MacOS a bit and I am solely treating it as an OS for editing videos & images on nowadays. [cite: 7] Outside of those use-cases I prefer not to use a Mac anymore. [cite: 8] I've never been a big Apple kinda guy, but still! [cite: 9]

Why I use a Tiling Window Manager [cite: 10]

I personally don't care whether a person uses a DE or a TilingWM. [cite: 11] In the end it's about you actually using the OS in a way that suits you. [cite: 12] For me, a tilingWM is all about 2 things: efficiency & simplicity. [cite: 13] The main argument against using a tilingWM is almost always "why make things harder for yourself?". [cite: 14] I never really understood this sentiment. Using a tilingWM is easier for me than using a DE nowadays because I'm used to it. [cite: 15] But as young teen I had never even heard of a tilingWM, or linux, or any of that stuff. [cite: 16] So the Windows XP DE was the easiest for me to use. [cite: 17] But this changed because I put in the effort to learn new things. [cite: 18] Frankly, and take this with a grain of salt because I'm biased, I think that if people would just put in the effort to learn a tilingWM workflow on their OS of choice, they wouldn't use "Why make things harder for yourself?" [cite: 19] as an argument anymore. The default isn't necessarily the easiest path. It's just the path you're most comfortable with. [cite: 20] Unfortunately to me, that's not much of an excuse.

I also like the simplicity tiling window managers bring. [cite: 21] Sure you can go all-out ricing something like hyprland on GNU/Linux, but I prefer the simple ones like DWM & i3. [cite: 22] Even on Windows I am noticing how a tiling window manager like GlazeWM makes most of the "Windows is pure bloat" arguments almost unnoticeable. [cite: 23] Under the hood these arguments are definitely true, but GlazeWM makes this bloat (almost) dissappear.

I like simplicity. [cite: 24] Maybe that's why Apple products never clicked with me? I don't need fancy animations or pretty buttons. [cite: 25] What I need is for it to work properly. That's all. [cite: 26] If I want to look at beautiful things I'll take a walk in nature. [cite: 27] PC's are turning into iPhones and tiling window managers are a direct way of going against this trend.
